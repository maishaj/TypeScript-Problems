# TypeScript — OOP & Type System

A quick reference guide covering core TypeScript concepts with practical examples.

---

## Topics Covered

### 🔷 Type System
- **`any` vs `unknown`** — Why `any` is a type safety hole and how `unknown` forces safe usage
- **Type Narrowing** — Using `typeof`, `instanceof`, and custom type guards
- **Generics** — Building reusable, strictly typed functions and components
- **`Pick` & `Omit`** — Creating specialized type slices while keeping code DRY

### 🔶 Object-Oriented Programming
- **Encapsulation** — Hide internal state, expose controlled interfaces
- **Inheritance** — Reuse and extend parent class logic
- **Abstraction** — Define contracts via abstract classes and interfaces
- **Polymorphism** — One interface, many behaviors

---

## Why TypeScript?

TypeScript helps manage complexity in large-scale projects by:

- Catching bugs at compile time, not runtime
- Making code self-documenting through types
- Enabling safe refactoring and scalability
